Bases: selfhost_client.base_client.BaseClient

A client for handling the timeseries section of NODA Self-host API

__init__(base_url: Optional[str] = None, username: Optional[str] = None, password: Optional[str] = None) → None

BaseClient constructor

Sets the base url for the NODA Self-host API and authentication credentials in a new requests session. Will look for the following environment variables if parameters are omitted:

SELF_HOST_BASE_URL

SELF_HOST_USERNAME

SELF_HOST_PASSWORD

Parameters

  • base_url (Optional[str]) – Base url to the NODA Self-host API.

  • username (Optional[str]) – Username for the NODA Self-host API. The domain name is the username in this case.

  • password (Optional[str]) – Password for the NODA Self-host API. The access key is the password in this case.

Raises

SelfHostFatalErrorException – The client could not find auth credentials and a base url for the NODA Self-host API

get_multiple_timeseries_data(uuids: List[str], start: str, end: str, unit: Optional[str] = None, ge: Optional[int] = None, le: Optional[int] = None, precision: Optional[str] = None, aggregate: Optional[str] = None, timezone: Optional[str] = None) → List[selfhost_client.types.timeseries_types.TimeseriesDataType]

Fetch multiple ranges of timeseries data from NODA Self-host API

Parameters

  • uuids (List[str]) – A series of timeseries UUIDs to search for.

  • start (datetime str) – Start (>=) of time period. The period (start to end) can not exceed 1 year. Must be in format: YYYY-MM-DDThh:mm:ss+hh:mm (as defined by RFC 3339, section 5.6.)

  • end (datetime str) – End (<=) of time period. The period (start to end) can not exceed 1 year. Must be in format: YYYY-MM-DDThh:mm:ss+hh:mm (as defined by RFC 3339, section 5.6.)

  • unit (optional[str]) – The SI unit of the result. A cast will occur if the base unit differs.

  • ge (optional[int]) – Value should be greater or equal to (>=) this.

  • le (optional[int]) – Value should be less or equal to (<=) this.

  • precision (optional[str]) – Truncate all timestamps and perform aggregate operations on the grouping. Available values : microseconds, milliseconds, second, minute, minute5, minute10, minute15, minute20, minute30, hour, day, week, month, year, decade, century, millennia

  • aggregate (optional[str]) –

    When using precision. Select this aggregate function instead of the default avg when computing the result. Does nothing when precision is not set. Available values:

    • avg

    • min

    • max

    • sum

    • count

  • timezone (optional[str]) – Act as this time zone. Defaults to UTC.
